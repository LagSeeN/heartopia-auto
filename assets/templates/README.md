# Templates (รูปปุ่ม/UI ที่บอทใช้หา)

โฟลเดอร์นี้เก็บไฟล์ `.png` เล็กๆ ที่ตัด crop มาจากหน้าจอเกมจริง
บอทจะเอารูปพวกนี้ไป "หา" บนหน้าจอด้วย image recognition แล้วคลิก

## วิธีสร้าง

รันเครื่องมือ capture (เปิดเกมค้างไว้ก่อน):

```
.venv\Scripts\python.exe -m tools.capture
```

แล้วลากเมาส์คร่อมปุ่มที่ต้องการ → กด ENTER → ตั้งชื่อตามตารางด้านล่าง

## ไฟล์ที่ต้องเตรียมสำหรับงาน "ทำอาหาร"

โหมดทำอาหารเป็นแบบ **"แตะทุกอย่างที่เห็น" รองรับหลายเตา** — บอทจะคลิก *ทุกจุด* บนหน้าจอ
ที่ตรงกับ template เหล่านี้ (กำหนดได้ใน `config.yaml` → `cooking.click_templates`)

> ⚠️ **สำคัญ: ต้อง capture จาก "หน้าเมนูทำอาหาร" (ตอนคุยกับเตาแล้ว) ไม่ใช่ตอนเดินอยู่เฉยๆ**
> เพราะปุ่มพวกนี้โผล่เฉพาะในเมนู — บอทจะกด `enter` (ตั้งใน `cooking.interact_key`) เปิดเมนูเตาให้เอง
> เมื่อยังไม่เจอปุ่มบนจอ แล้วค่อยเริ่มคลิกปุ่มที่ capture ไว้

| ชื่อไฟล์ | คือรูปอะไร | จำเป็น |
|---|---|---|
| `stove.png` | **ตัวเตา (ตอนเดิน) ที่คลิกเพื่อเปิดเมนูทำอาหาร** — ขั้นแรกสุดของ flow! | ✅ ต้องมี |
| `cook_start.png` | ปุ่ม "ปรุง" ในเมนู (ปุ่มสีฟ้า/เขียว) | ✅ ต้องมี |
| `heat_adjust.png` | ปุ่ม/ไอคอน "ปรับระดับไฟ" (Stove Heat) — บอทกดให้อัตโนมัติทุกครั้งที่เห็น | ✅ ต้องมี |
| `cook_done.png` | **ปุ่มเก็บอาหาร / แตะรับผลลัพธ์** ตอนปรุงเสร็จ (ต้องเป็น *ปุ่มที่กดเพื่อเก็บ* ไม่ใช่แค่รูปอาหาร) | ✅ ต้องมี |
| `cook_again.png` | ปุ่ม "ทำอาหารต่อ" (ถ้าเกมมี — ถ้าไม่มีก็วนด้วยการคลิก `stove` ใหม่) | ⬜ เสริม |

> ✅ **flow ที่บอททำ:** คลิก `stove` เปิดเมนู → คลิก `cook_start` → กด `heat_adjust` → กด `cook_done` เก็บอาหาร
> → เมนูปิด เห็น `stove` อีก → คลิกเปิดใหม่ → **วนเอง** (จุดสำคัญที่เคยขาดคือ `stove` — ถ้าไม่มี บอทจะเปิดเมนูไม่ได้)

> 💡 **ถ้าบอทไม่ยอมเก็บอาหาร:** มักเป็นเพราะ `cook_done.png` ตัดมาจาก *รูปอาหาร/ข้อความ*
> ที่ไม่ใช่ปุ่มกด — ให้ capture ใหม่ที่ **ปุ่มจริงที่ต้องแตะเพื่อเก็บ**

> 🔸 capture `stove` ให้ครอบส่วนที่เด่นของเตา (เช่น หัวเตา/เตาอบ) ตอนยืนหน้าเตา — ระวังอย่าให้เหมือนเฟอร์นิเจอร์อื่น

> 🔥 **หลายเตา:** ถ้าหลายเตาแสดงปุ่มเดียวกัน บอทจะคลิกครบทุกเตาในสแกนเดียว
> (ปรับระยะแยกเตาได้ที่ `cooking.dedup_distance`) — ไม่ต้องตั้งจำนวนหรือลำดับใดๆ เอง

> เคล็ดลับ: ตัดให้ครอบเฉพาะส่วนที่ "ไม่เปลี่ยน" ของปุ่ม (เช่น ไอคอน/ตัวอักษร)
> อย่าตัดกว้างจนติดพื้นหลังที่เปลี่ยนได้ จะ match แม่นกว่า
