# Heartopia Auto 🍳

โปรแกรมช่วยเล่นอัตโนมัติสำหรับเกม **Heartopia** (PC – Steam/TapTap) แนวเดียวกับ
[ok-nte](https://github.com/BnanZ0/ok-nte) / [ok-script](https://github.com/ok-oldking/ok-script)
คือทำงานด้วยการ **จับภาพหน้าจอ → หาปุ่ม/UI ด้วย image recognition (OpenCV) → สั่งคลิกอัตโนมัติ**
โดย **ไม่ยุ่ง/ไม่แก้ไฟล์เกม** เลย เห็นอะไรบนจอก็ทำเหมือนคนนั่งคลิกเอง

งานที่ทำได้ตอนนี้: **ทำอาหาร (Cooking)** — และต่อยอดงานอื่น (ตกปลา/ทำสวน) ได้ง่าย

> ⚠️ **อ่านก่อนใช้:** การใช้บอท/automation อาจผิดเงื่อนไขการให้บริการ (ToS) ของเกม
> และ **มีความเสี่ยงโดนแบนบัญชี** ใช้ด้วยความรับผิดชอบของตัวเอง แนะนำให้ทดสอบกับบัญชีรอง
> โปรเจกต์นี้ทำเพื่อการศึกษา/ใช้ส่วนตัวเท่านั้น

## 1. ติดตั้ง (ทำครั้งเดียว)

มี Python อยู่แล้ว และผมสร้าง virtualenv + ลงไลบรารีให้เรียบร้อยแล้วในโฟลเดอร์ `.venv`
ถ้าต้องลงใหม่ (เช่นย้ายเครื่อง):

```powershell
cd "C:\Users\danupol.intranurux\Git Project\Other\heartopia-auto"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## 2. Calibrate — capture รูปปุ่มจากเกม (สำคัญที่สุด!)

บอทยังไม่รู้จักหน้าตาปุ่มในเกมของคุณ ต้องสอนมันก่อนโดยตัดรูปปุ่มมาเก็บไว้

1. เปิดเกม Heartopia ไปที่ **หน้าทำอาหาร (เตา)**
2. รันเครื่องมือ capture:
   ```powershell
   .\.venv\Scripts\python.exe -m tools.capture
   ```
3. เลือกหน้าต่างเกม → กด ENTER เพื่อจับภาพ → **ลากเมาส์คร่อมปุ่ม** → กด ENTER → ตั้งชื่อไฟล์
4. capture ให้ครบตามตารางใน [`assets/templates/README.md`](assets/templates/README.md)
   อย่างน้อยต้องมี: **`stove`** (ตัวเตาตอนเดิน — คลิกเปิดเมนู!), `cook_start`, `heat_adjust`, `cook_done`
   > ⚠ ต้องมี `stove` ด้วย ไม่งั้นบอทเปิดเมนูทำอาหารเองไม่ได้ (จะไม่เริ่มทำ/ทำครั้งเดียว)
   > ⚠ `cook_done` ต้องเป็น **ปุ่มที่กดเพื่อเก็บอาหารจริง ๆ** (ไม่ใช่แค่รูปอาหาร)

## 3. รันโปรแกรม

```powershell
.\.venv\Scripts\python.exe run.py
```

หน้าต่างควบคุมแบบ **Fluent Design (เหมือน ok-nte)** จะเปิดขึ้น — มี sidebar (ควบคุม / ตั้งค่า),
ธีม dark/light/auto, การ์ด, สวิตช์ และรองรับ 2 ภาษา (เปลี่ยนภาษา/ธีมได้ในหน้า **Settings**, ค่าเริ่มต้น English):
1. **เลือกหน้าต่างเกม** (มันเดาให้อัตโนมัติ ถ้าไม่ถูกกดรีเฟรชแล้วเลือกเอง)
2. กด **🔍 ทดสอบการตรวจจับ** ก่อน — ดูว่าบอทเจอปุ่มที่ capture ไว้ครบไหม (ดู score ควร > 0.80)
   จะมีภาพผลลัพธ์เซฟไว้ใน `assets/screenshots/`
3. ตั้งค่า (จะหยุดเมื่อได้กี่จาน / หน่วงสแกน) → กด **▶ Start**
4. ต้องการหยุด: กด **■ Stop**, กด **F9**, หรือ **ดันเมาส์ชนมุมซ้ายบนสุดของจอ**

> 🔥 **โหมดทำอาหาร = "แตะทุกอย่างที่เห็น" รองรับหลายเตา** — บอทสแกนแล้วคลิก *ทุกจุด* ที่เจอ
> `stove` (เปิดเมนู) / `cook_start` / `heat_adjust` / `cook_done` → flow ครบลูป ทำหลายเตาได้
> และ **ไม่ต้องตั้งจำนวนปรับไฟเอง** (เห็นปุ่มปรับไฟก็คลิกให้อัตโนมัติ)

## 4. ปรับแต่ง

แก้ค่าได้ใน [`config.yaml`](config.yaml) เช่น:
- `vision.match_threshold` — ถ้าหาปุ่มไม่เจอลองลดเหลือ `0.75`; ถ้าเจอมั่ว/ผิดให้เพิ่มเป็น `0.88`
- `cooking.repeat` — `0` = ทำจนกดหยุด ; `>0` = หยุดเมื่อเก็บอาหารครบ N จาน
- `cooking.scan_delay` — หน่วงระหว่างสแกนหน้าจอ (น้อย = ไวแต่กิน CPU)
- `cooking.click_cooldown` — กันคลิกจุดเดิมซ้ำเร็วเกิน (วินาที), `0` = คลิกรัวทุกสแกน
- `cooking.dedup_distance` — ระยะ (px) ที่ถือว่าเป็นจุด/เตาเดียวกัน
- `cooking.click_templates` — รายชื่อปุ่มที่บอทจะคลิกทุกจุดที่เจอ (รวม `stove` = ตัวเตา)
- `cooking.interact_key` — (สำรอง) กดปุ่มคีย์บอร์ดเปิดเมนูเตา ค่าเริ่มต้นปิดไว้ `""` (ปกติใช้ template `stove` คลิกเปิดดีกว่า)
- `language` — `en` / `th` ภาษาเริ่มต้นของ UI (เปลี่ยนได้ในหน้า Settings)
- `theme` — `auto` / `dark` / `light` ธีมหน้าตา (เปลี่ยนได้ในหน้า Settings)
- `input.instant_move` — `true` = เมาส์พุ่งไปจุดเป้าหมายทันที (ไม่ลากไล่)
- `input.action_delay` — หน่วงหลังทุกครั้งที่คลิก (รอ UI เกมตอบสนอง) เพิ่มถ้าเกมตอบช้า
- `runtime.background_mode` — `true` = **โหมดเบื้องหลัง** พับจอ/สลับไปทำอย่างอื่นได้ (เท่ากับติ๊ก checkbox)
- `vision.scales: [0.9, 1.0, 1.1]` — เปิดถ้าความละเอียดจอต่างจากตอน capture

### โหมดเบื้องหลัง (เหมือน ok-nte)

ติ๊ก **"โหมดเบื้องหลัง"** ใน GUI (หรือตั้ง `runtime.background_mode: true`) จะจับภาพด้วย `PrintWindow`
และสั่งคลิกด้วย `PostMessage` ตรงเข้าหน้าต่างเกม — **ไม่แย่งเมาส์จริง** สลับไปทำงานอื่นได้

- ✅ ไม่ต้องเอาเกมขึ้นหน้าสุด, ไม่ขยับเมาส์เรา
- ⚠ **best-effort** — บาง engine เกมไม่รับคลิกแบบนี้ ถ้าคลิกแล้วเกมไม่ตอบสนอง ให้ปิดโหมดนี้
- ⚠ **อย่าย่อ (minimize) หน้าต่างเกม** — ปล่อยให้เปิดอยู่ข้างหลังได้ ไม่งั้นจับภาพเป็นจอดำ
- 💡 แนะนำกด **🔍 ทดสอบการตรวจจับ** ในโหมดนี้ก่อน เพื่อเช็กว่าจับภาพ + เจอปุ่มได้จริง

> ⚠️ **กรณี Heartopia (เวอร์ชัน PC Steam/TapTap):** ทดสอบแล้วพบว่า *จับภาพได้* แต่ *คลิกไม่เข้า*
> เพราะตัวเกม (engine Unity) ไม่รับคลิกผ่าน PostMessage → **ให้ใช้โหมด foreground** (ค่าเริ่มต้น)
> ถ้าอยากพับจอทำอย่างอื่นจริง ๆ ต้องเล่นผ่าน **Android emulator แล้วใช้ ADB** (ขอให้แจ้งถ้าต้องการให้เพิ่ม backend นี้)

## โครงสร้างโปรเจกต์

```
heartopia-auto/
├── run.py                  # จุดเริ่มโปรแกรม (เปิด GUI)
├── config.yaml             # ตั้งค่าทั้งหมด
├── requirements.txt
├── tools/
│   └── capture.py          # เครื่องมือ capture template (calibrate)
├── src/
│   ├── window.py           # หา/โฟกัสหน้าต่างเกม + DPI
│   ├── capture.py          # จับภาพหน้าจอ (mss)
│   ├── vision.py           # image recognition (OpenCV template matching)
│   ├── input_controller.py # คลิก/ลาก/กดปุ่ม + failsafe
│   ├── bot.py              # engine รวมทุกอย่าง + helper (find/wait/click/sleep)
│   ├── i18n.py             # ระบบ 2 ภาษา (English / ไทย)
│   ├── task.py             # คลาสฐานของงาน
│   ├── gui_qt.py           # หน้าต่างควบคุมแบบ Fluent (PySide6 + Fluent-Widgets) ← หลัก
│   ├── gui.py              # หน้าต่างควบคุมสำรอง (Tkinter) ใช้เมื่อไม่มี PySide6
│   └── tasks/
│       ├── __init__.py     # ทะเบียนงานทั้งหมด
│       └── cooking.py      # งานทำอาหาร
└── assets/
    ├── templates/          # รูปปุ่มที่ capture ไว้ (.png)
    └── screenshots/        # ภาพผลการตรวจจับ
```

## เพิ่มงานใหม่ (เช่น ตกปลา)

1. สร้าง `src/tasks/fishing.py` สืบทอด `BaseTask`, เขียนเมธอด `run(self, bot)`
   ใช้ helper เช่น `bot.wait_for("...")`, `bot.click("...")`, `bot.sleep(...)`
2. เพิ่มใน `get_all_tasks()` ที่ [`src/tasks/__init__.py`](src/tasks/__init__.py)
3. capture template ปุ่มที่เกี่ยวข้อง แล้วรันได้เลย — มันจะโผล่ใน dropdown ของ GUI

## แก้ปัญหาที่พบบ่อย

| อาการ | วิธีแก้ |
|---|---|
| **`Windows บล็อกการสั่งเมาส์` / `SetCursorPos` error** | ตัวเกมรันแบบ Admin → เปิดบอทด้วย **`run_as_admin.bat`** (หรือคลิกขวา → Run as administrator) |
| คลิกแล้วเมาส์ขยับแต่เกมไม่ตอบ / จับภาพเป็นจอดำ | ตั้งเกมเป็นโหมด **Windowed / Borderless** (อย่าใช้ Exclusive Fullscreen) |
| หาหน้าต่างเกมไม่เจอ | กดรีเฟรชแล้วเลือกเอง / เพิ่มคำใน `window.title_keywords` |
| คลิกผิดตำแหน่ง | ตรวจว่าเล่นแบบ windowed และไม่ย่อหน้าต่าง; capture template ใหม่ที่ความละเอียดปัจจุบัน |
| หาปุ่มไม่เจอ (score ต่ำ) | ตัด template ใหม่ให้แคบลงเฉพาะไอคอน/ตัวอักษร; ลด `match_threshold` |
| คลิกแล้วเกมไม่ตอบสนอง | เกมต้องอยู่ foreground; บอทจะดึงหน้าต่างขึ้นมาให้เอง อย่าคลิกแทรกระหว่างทำงาน |
```
